# Milestone v1.3 Audit: Brazil Commercial Listing Map

**Status:** gaps_found

## Integration Findings

**Wired:**

- Phase 11 uses Phase 10 `upsertListing` and `upsertListingImportTarget` helpers.
- Listing dedupe is backed by `(user_id, source, source_url)` upsert conflict.
- Import runs are created, completed, failed, shown in the import page, and linked to run detail pages for new runs.
- Auth gating exists at layout and server action level.
- Sidebar exposes the listing import surface.

**Broken or partial:**

- No form/action exists to create arbitrary saved OLX import targets by state/city/search term.
- Duplicate accounting is incomplete because successful upserts are counted as created records.
- Classification, geocoding, map display, and MVP review flows are absent.

## Broken Flows

| Flow | Affected Requirements | Break |
|------|-----------------------|-------|
| Custom configured OLX target import | DATA-03, SRC-01 | No UI/action creates user-defined saved `listing_import_targets`. |
| Import status with skipped duplicates | SRC-04 | `updatedCount` and `skippedCount` remain `0`; duplicate upserts count as created. |
| Classification pipeline | CLS-01, CLS-02, CLS-03, CLS-04 | No classifier action/job writes commercial fields. |
| Geocoding pipeline | GEO-01, GEO-02 | No geocoder action/job writes coordinates. |
| Opportunity map and review | MAP-01, MAP-02, MAP-03, QA-01, QA-02 | No map route/components or listing review flow exists. |

## Tech Debt

- Phase 11 summary artifacts still mention the removed Facebook/manual CSV implementation.
- On-demand location input is passed as both `region` and `city`, duplicating OLX query terms.
- Run detail page depends on `metadata.savedUrls`, so older runs cannot show exact listings.

## Recommendation

Do not archive v1.3 yet unless the milestone scope is intentionally reduced to an ingestion-only release. The practical next step is to plan and execute Phase 12, then Phase 13.
