## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, ring tick/gap details, compact list spacing |
| sm | 8px | Badge groups, category bar label/value spacing, list item gaps |
| md | 16px | Default card internal groups and stacked mobile sections |
| lg | 24px | Score card padding, card header/content separation |
| xl | 32px | Desktop split between score summary and breakdown/list content |
| 2xl | 48px | Major section separation only if the score block becomes its own page band |
| 3xl | 64px | Not used in Phase 18 component internals |

Exceptions: Interactive controls keep existing primitive heights: `SelectTrigger` 44px, default `Button` 40px, small `Button` 36px, icon controls 36px. The circular score ring is fixed at 112px on mobile and 128px on `sm` and up so content does not shift when score digits change.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#050505` / `bg-background` | Page background and dominant surrounding surface |
| Secondary (30%) | `#0b0b0b` / `bg-card`, `#111111` / `bg-secondary` | Score card, embedded panels, category bar tracks, empty-state panel |
| Accent (10%) | Score-band semantics listed below; default accent token `#1d1d1d` stays for hover/focus surfaces only | Score ring, score band badge, category bar fill, signal/risk emphasis |
| Destructive | `#ff6868` / `text-destructive` | High-risk flags and actual destructive actions only |

Accent reserved for: score ring progress, total score band, fit label badge, category bar fills, positive signal markers, risk severity markers, active strategy selection, and primary compute/recompute CTA.

Score band colors:
- Green `#22c55e`: total score >= 70, fit label `Forte`, positive signal markers.
- Yellow `#eab308`: total score >= 50 and < 70, fit label `Moderado`.
- Orange `#f97316`: total score >= 40 and < 50, low-score warning and medium risk.
- Red `#ff6868`: total score < 40, fit label `Fraco`, high risk.

Use color plus text labels. Do not rely on color alone to communicate score quality, signal polarity, or risk severity.

## Phase 18 Visual Contract

### Layout

The score UI is one compact section directly below the location insight section:

1. Header row:
   - Left: eyebrow `Pontuação de oportunidade`, heading `Score por estratégia`.
   - Right: strategy selector and compute/recompute action. On mobile, stack selector above the button.
2. Main score card:
   - Left/top: circular score ring with large numeric score and fit badge.
   - Right/below: selected strategy label, score metadata, and five category bars.
   - Bottom: two lists, `Pontos fortes` and `Riscos`, side by side on `sm` and up, stacked on mobile.

Use a scan-first hierarchy. The broker must see total score, strategy, fit label, all five bars, top positives, and top risks without opening a details toggle.

### Circular Score Ring

- Size: 112px mobile, 128px `sm` and up.
- Stroke/ring thickness: 6px.
- Track: `border` or `muted`.
- Progress: score band color from the Color section.
- Center content: score number in 40px semibold, small `/100` label in 12px muted text.
- Accessible label: `Score {totalScore} de 100 para {strategyLabel}, fit {fitLabel}`.

### Category Bars

Always render the five categories in this fixed order:

| Engine category | Portuguese label |
|-----------------|------------------|
| `demographics` | `Demografia` |
| `locationQuality` | `Qualidade da localização` |
| `nearbyBusinesses` | `Fluxo potencial` |
| `competition` | `Concorrência` |
| `risk` | `Risco` |

Per Phase 18 context D-03, the scan-first card renders five visible bars. If the engine payload includes `investorFit`, do not add a sixth default bar in this first implementation; its impact remains reflected in the total score and signal/risk copy. Each bar row uses:
- 12px label, 12px numeric score aligned right.
- 8px high track with `rounded-full`.
- Filled width clamped between 0% and 100%.
- Fill color follows the same score band thresholds as total score.

### Signals and Risks

- Show at most three positive signals and three risks in the default card.
- Positive signals use green marker and plain Portuguese label.
- Risks use orange for `medium`/`low`, red for `high`, and include severity text only when available.
- If a list is empty, render muted fallback copy: `Sem sinais suficientes.` or `Sem riscos destacados.`

### Empty States

Missing `locationInsight`:
- Render the score card region as an empty state, not an error.
- Required copy: `Enriqueça a localização antes de calcular a pontuação`.
- Provide an action path to the existing location enrichment flow. Do not invent a new route.

No saved score for selected strategy:
- Keep the strategy selector visible.
- Show CTA `Calcular pontuação`.
- Do not show fake score, skeleton bars, or neutral score defaults.

Action error:
- Keep the previously saved score visible if one exists.
- Show the Portuguese error message from `ScoringActionState.errors.general[0]` when available.
- Fall back to the Error state copy in this spec only when the action returns no message.

## Interaction Contract

| Interaction | Required Behavior |
|-------------|-------------------|
| Select strategy | Update selected strategy locally. If a saved score for that strategy is already loaded, show it. If none exists, show the compute CTA. |
| Click `Calcular pontuação` | Call `scoreListingAction(listingId, strategySlug)` inside `useTransition`; disable selector/button while pending; show spinner and `Calculando...`. |
| Click `Recalcular pontuação` | Same action path; on success, refresh route and show updated score/version. |
| Action success | Render returned score immediately if available, then `router.refresh()` to sync server-rendered data. |
| Missing enrichment | Do not call scoring automatically. Point broker to existing enrichment action first. |
| Keyboard | `Select` and buttons must be reachable by tab, visible focus ring from shadcn primitives retained. |

No destructive actions are introduced in Phase 18.
