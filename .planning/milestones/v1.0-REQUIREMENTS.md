# RealTools — v1 Requirements

## Authentication

- [x] **AUTH-01**: User can create an account with email and password
- [x] **AUTH-02**: User can log in with email and password and stay logged in across sessions
- [x] **AUTH-03**: User can log out from any page

## Deal Management

- [x] **DEAL-01**: User can create a deal with title, address, price, and description
- [x] **DEAL-02**: User can view, edit, and delete their own deals
- [x] **DEAL-03**: User can set deal status: active, negotiating, or closed
- [x] **DEAL-04**: User sees a dashboard listing all their deals with status and a New Deal button

## Notes

- [x] **NOTE-01**: User can add a note to a deal
- [x] **NOTE-02**: User can edit their own notes on a deal
- [x] **NOTE-03**: User can delete their own notes on a deal

## File Uploads

- [x] **FILE-01**: User can upload files to a deal (stored in Supabase Storage)
- [x] **FILE-02**: User can view and download uploaded files from the Deal Hub

## Offering Memorandum

- [x] **OM-01**: User can generate a hosted OM HTML page from deal data and uploaded images
- [x] **OM-02**: OM page is publicly accessible at `/om/[deal-id]` without requiring auth
- [x] **OM-03**: OM page includes deal title, property details, description, and images

## Buyers CRM

- [x] **BUYER-01**: User can create a buyer with name, email, and tags
- [x] **BUYER-02**: User can view and edit buyers
- [x] **BUYER-03**: User can delete buyers
- [x] **BUYER-04**: User can manually select buyers to associate with a deal

## Send OM

- [x] **SEND-01**: User can send the OM link via email to selected buyers using Resend
- [x] **SEND-02**: Each buyer receives a unique OM URL with a tracking token (`/om/[deal-id]?ref=[token]`)

## Open Tracking

- [x] **TRACK-01**: System records per-buyer OM open when buyer visits the OM URL
- [x] **TRACK-02**: OM page embeds a tracking pixel (`/api/track/[token]`) as a secondary open signal
- [x] **TRACK-03**: Each open event is idempotent

## Activity Log

- [x] **ACT-01**: Deal Hub shows a per-deal activity log with OM sent, note added, and file uploaded events
- [x] **ACT-02**: Activity log shows per-buyer OM open events

## Deferred Verification Debt

- Phase 3 implementation is complete, but live Resend delivery and browser-driven tracking/activity confirmation remain pending in `03-HUMAN-UAT.md`.
